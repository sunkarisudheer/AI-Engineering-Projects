# Assignment 16 — JSON Prompting for API Reliability

## Task

Create one prompt that returns reliable JSON for an application workflow and run it locally.

## Rules

Your prompt must include:
1. A clear role
2. A clear task
3. A fixed JSON structure
4. At least 3 fixed keys
5. At least 1 field with limited allowed values
6. A successful JSON parse in Python

## Enterprise or operational use cases to choose from

Choose any one:
- e-commerce return triage assistant
- cyber alert classification assistant
- hospital intake routing assistant
- logistics issue triage assistant
- insurance claim routing assistant

## Submission

Push your code to your GitHub repository and share the public repo link.

## What will be checked

- correct understanding of JSON reliability
- clear fixed structure
- realistic controlled values
- valid JSON parsing
- clean code
- working output
