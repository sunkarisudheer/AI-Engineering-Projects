# Assignment 13_07 — Input Guardrails

## Task

Create one workflow that checks the input before calling the model.

## Rules

Your code must include:
1. At least 3 required input fields
2. At least 1 validation rule for an allowed value
3. A blocked case where the model should not be called
4. A valid case where the model is called
5. Clear printed validation messages

## Enterprise use cases to choose from

Choose any one:
- medical device support intake assistant
- emergency hotline intake assistant
- gaming abuse report intake assistant
- military equipment maintenance intake assistant
- workplace incident reporting intake assistant

## Submission

Push your code to your GitHub repository and share the public repo link.

## What will be checked

- correct understanding of input guardrails
- simple and clear validation logic
- proper block-or-continue behavior
- clean code
- working output
