# Assignment 17 — Safe Prompt Workflow with Constraints & Fallbacks

## Task

Create one safe prompt workflow that includes constraints and fallback behavior, then run it locally.

## Rules

Your prompt must include:
1. A clear role
2. A clear task
3. At least 4 constraints
4. At least 3 fallback rules
5. A structured final output
6. A realistic high-trust or safety-sensitive use case

## Enterprise or critical-domain use cases to choose from

Choose any one:
- emergency assistance routing assistant
- hospital triage intake assistant
- aviation incident routing assistant
- military field support routing assistant
- industrial safety response assistant

## Submission

Push your code to your GitHub repository and share the public repo link.

## What will be checked

- correct understanding of safe workflow design
- clear constraints
- realistic fallback behavior
- useful structured response
- clean code
- working output
