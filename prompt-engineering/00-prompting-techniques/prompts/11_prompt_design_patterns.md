# 11 — Prompt Design Patterns

## What it means

Prompt design patterns are common prompt structures used to solve different kinds of tasks.

They are not the same as prompting techniques like zero-shot or few-shot.

## Simple difference

- Prompting technique = how you prompt
- Prompt design pattern = what task the prompt is solving

## 4 beginner-friendly prompt design patterns

### 1. Classification
Used when you want the model to assign a label.

Examples:
- sentiment classification
- topic classification
- priority tagging

### 2. Extraction
Used when you want the model to pull specific information from text.

Examples:
- names
- dates
- key points
- product details

### 3. Transformation
Used when you want the model to rewrite or convert text.

Examples:
- simplify
- formalize
- translate
- shorten

### 4. Generation
Used when you want the model to create new content.

Examples:
- reply writing
- idea generation
- email drafting
- plan creation

## Why this matters

Design patterns help you think more clearly about what kind of output you actually need.

## Key idea

Before writing a prompt, first ask:
What task type is this?
